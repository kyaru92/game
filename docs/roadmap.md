# 当前状态与路线图

本文档用于记录项目当前能力、主要限制和后续迭代方向。它不是详细需求文档，只帮助后续开发判断“现在有什么、下一步该往哪里放”。

## 1. 当前状态

项目当前是一个 Canvas ECS MVP，重点验证架构和玩法基础模块。

已经具备：

- React + Canvas 的运行界面。
- 玩家自由移动。
- 实体、位置、碰撞、生命值、属性。
- 背包、快捷栏、装备。
- 物品激活、施法、冷却、次数消耗。
- effect、叠层、持续时间、周期结算、属性修正。
- 直接伤害、范围伤害。
- 枪械、弹药、装填。
- 投射物发射、飞行、命中、穿透、impact 结算。
- 传送道具。
- 实体生成道具。
- 调试指令与指令补全。
- JSONC 数据配置与校验。

## 3. 当前限制

当前项目仍是 MVP，因此有一些刻意简化的部分：

- 没有正式地图系统，世界尺寸和障碍主要在 runtime 中初始化。
- 没有完整 AI 行为系统。
- 没有存档 / 读档。
- 没有关卡流程、胜负条件或任务系统。
- UI 和 Canvas 绘制集中在 `App.tsx`，后续变复杂时可能需要拆分。
- 自动化测试不足，目前主要依赖 TypeScript 编译和数据校验。
- 数据字段文档不是逐字段手册，字段细节以 schema 为准。

## 4. 后续迭代方向

优先级可按项目需要调整。

### 架构维护类

- 拆分 `App.tsx` 中的 Canvas 绘制、HUD、背包、指令栏等 UI 模块。
- 为关键 system 增加单元测试。
- 让 `World` 保持基础状态容器定位，避免继续堆积单一玩法规则。
- 为事件名和事件 payload 建立更明确的类型约束。

### 数据与工具类

- 扩充 `validate-data.ts` 的跨引用和互斥校验。
- 为常用 component 增加更清晰的示例数据。
- 如数据增长，考虑拆分数据目录，例如 `data/effects/*.jsonc`。
- 提供数据配置模板或生成脚本。

### 玩法类

- 增加 AI system。
- 增加地图 / 场景 system。
- 增加阵营、仇恨、感知、寻路等基础战斗规则。
- 增加装备槽、被动属性、掉落和拾取。
- 增加技能或更复杂的 item activation 组合。
- 增加任务、波次、胜负条件。

### 表现类

- 抽离并扩展视觉事件。
- 增加更完整的动画、音效、命中特效。
- 改善 UI 信息密度和调试面板。

## 5. 迭代注意事项

后续做功能时，优先遵守以下判断：

1. **是数据字段变化吗？**  
   先改 schema，再改 system，再改数据。

2. **是新玩法规则吗？**  
   优先放到 system，不放到 UI。

3. **是物品激活后的结果吗？**  
   优先通过事件接入 `OnItemActivation`。

4. **是持续更新行为吗？**  
   实现 `update()`，并在 `runtime.ts` 注册到 `world.systems`。

5. **是展示变化吗？**  
   优先修改 UI；如果需要复杂摘要，放到纯函数或引擎工具中。

6. **是调试便利吗？**  
   可以扩展 command，但不要让 command 成为正式玩法规则的唯一实现。
